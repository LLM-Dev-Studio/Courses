# Module 1 Overview: Introduction & Core Concepts

Before writing a single line of code or clicking a single button, you need a mental model of what AI Dev Studio is and why it exists. This module builds that foundation.

## What you will learn

- The problem AI Dev Studio solves and why single-agent LLMs are not enough
- The domain vocabulary used throughout the platform (Agent, Task, Decision, Executor, Session)
- How the major components relate to each other architecturally

## Lessons in this module

1. What is AI Dev Studio?
2. Core Concepts
3. Architecture Overview
4. Module Quiz

> [!NOTE]
> The terms introduced here appear everywhere — in the UI labels, in the codebase, and in this course. Learning them now will make every later module click faster.

## Key Takeaway

A clear mental model of the domain — Agents, Tasks, Decisions, Executors, Sessions — is the single most valuable thing you can take from this module.
